# Plan-First Workflow
## Rules
- NEVER write code, create files, or run commands before a plan is approved.
- NEVER skip steps.
- If new work is discovered, add it to the plan and ask for approval again.

## Workflow
1. Analyze the project.
2. Propose a `PLAN.md` file.
3. Stop and ask for approval.
4. Execute only after approval.
